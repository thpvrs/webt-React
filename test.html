ก่อนที่เราจะไปทำความรู้จักกับ REACT ว่ามันคืออะไร เราจะพามาดูกันก่อนว่า
การทำเว็บแบบเดิมที่เราทำมันต่างจากการทำเว็บแบบใหม่ด้วย React ยังไง

การทำเว็บไซต์แบบดั้งเดิม

        การทำเว็บไซต์ด้วยเทคนิคเดิม คือ HTML, CSS, JavaScript, PHP (หรือภาษา Backend ภาษาใดภาษาหนึ่ง) ส่วนใหญ่ “การทำงานทั้งหมดจะอยู่ที่ Server” สามารถสรุปลำดับการทำงานได้ดังนี้

1. User เข้าเว็บไซต์เรา https://www.designil.com ก็จะส่งไปบอก Server
2. Server (PHP) จะประมวลผลผ่านภาษา Backend ว่า User เข้าหน้านี้แล้วต้องให้อะไรบ้าง เช่น ต้องการบทความ 10 บทความล่าสุด
3. Server นำข้อมูลที่เก็บไว้ออกมาจาก Database (เช่น MySQL) แล้ววางลงไปในเทมเพลต HTML ที่เตรียมไว้
4. Server ส่ง HTML กลับไปหา User

การทำเว็บไซต์แบบใหม่

         การทำเว็บไซต์แบบใหม่ คือ ใช้ JavaScript ส่ง Request ไปขอข้อมูลจาก Server แล้ว Server ตอบกลับมาเป็นข้อมูลอย่างเดียว เราค่อยใช้ JavaScript เอาข้อมูลมาแสดงอีกที อย่างไรก็ตาม พอข้อมูลถูกส่งจาก Server มาให้ Web Browser แล้ว ก็เกิดปัญหาว่า “แล้วจะจัดการข้อมูลบน Web Browser อย่างไร” ซึ่งเมื่อก่อนไม่มี Library คอยช่วยเหลือด้านการแสดงผล เลยทำให้โค้ดพันกันมั่วไปหมด
        JavaScript Framework สมัยใหม่ เช่น React ก็เลยออกมาเพื่อช่วยให้เราจัดการเรื่องการดึงข้อมูล การแสดงผลข้อมูลในเว็บให้ง่ายขึ้น

1. User เข้าเว็บไซต์เรา
2. โหลด HTML, CSS, JS ขึ้นมาก่อน แล้ว JavaScript ค่อยขอบทความล่าสุดจาก Server
3. Server ได้รับคำขอ ก็ส่งไปขอข้อมูลจากฐานข้อมูลอีกที

4. Database ส่งข้อมูลกลับมาให้ Server
5. Server ส่งข้อมูลกลับไปให้ Browser ทาง API โดยส่วนใหญ่จะส่งเป็น JSON เพราะ JavaScript สามารถนำไปใช้งานต่อได้ง่าย
6. JavaScript นำข้อมูลที่ได้รับมาจัดการแสดงผลบนหน้าเว็บไซต์






React คืออะไร ?

                 React เป็น JavaScript library หรือจะเรียกว่าเป็น Javascript Framework เป็น JavaScript Front-End Frameworks ที่พัฒนาโดย Facebook ที่ใช้สำหรับสร้าง user interface ที่ให้เราสามารถเขียนโค้ดในการสร้าง UI ที่มีความซับซ้อนแบ่งเป็นส่วนเล็กๆออกจากกันได้ ซึ่งแต่ละส่วนสามารถแยกการทำงานออกจากกันได้อย่างอิสระ และทำให้สามารถนำชิ้นส่วน UI เหล่านั้นไปใช้ซ้ำได้อีก หรือจะพูดว่าเราใช้ React เพื่อทำให้เราสามารถสร้าง

"ภาพ(view)"

นั่นเอง




    จากแต่ก่อนเวลาเราจะเขียนหน้าตาของเว็บ เราก็จะทำผ่าน HTML เช่น




    แต่ใน React การจะสร้างหน้าตาของเว็บขึ้นมาได้นั้น มันมาจากสิ่งที่เรียกว่า Component ให้นึกภาพว่า Component เป็น block ส่วนย่อยของเว็บเราที่จะสร้างออกมา โดยเราจะเขียนโค้ดภาษา JavaScript เพื่ออธิบายหน้าตา (อาจรวมถึงพฤติกรรม) ของ block นั้นๆ แทนที่เราจะเขียนด้วย HTML จากรูปข้างบนเมื่อเขียนด้วย React จะได้


    ซึ่ง ReactDOM.render() เป็นฟังก์ชันที่ตัว React จะทำการ render HTML ที่เขียนมา โดยมีจุดที่จะ render อยู่ที่ DOM ที่มี id เป็น root จากตัวอย่างนี้

ถ้าเรามี HTML
ตัว React จะทำการ render หน้า HTML ทั้งหมดเป็น


    นั่นคือเราจะไม่เขียน HTML ใน HTML กันอีกต่อไป เราจะใช้ React เขียนอยู่ในรูปแบบ JavaScript แทน ซึ่งทำให้มีภาษาใหม่เกิดขึ้นมาคือ JSX ที่ทำให้เรายัด HTML เข้าไปเขียนใน JavaScript ได้ ดังนั้น React คือการที่เราจะเขียน View ด้วยภาษา JavaScript ที่มี HTML แทรกอยู่ ก็คือ JSX นั่นเอง

    สรุปคือ

สำหรับการเขียน Component นั้น ก็เหมือนกับการเขียน HTML ส่วน React ใช้สิ่งที่เรียกว่า JSX ในการแสดงผลเว็บไซต์หน้าตาจะเหมือน HTML แตกต่างตรงเราเขียนเข้าไปในไฟล์ JavaScript แทนไฟล์ HTML ทำให้เราสามารถเล่นอะไรกับมันได้มากกว่า

อีกตัวอย่างการเปรียบเทียบระหว่าง HTML กับ React
วิธีคิดของ React คือ ทุกอย่างเป็น Component



ถ้าเราอยากเขียน React
เราต้องเตรียมตัวและมีพื้นฐานยังไงบ้าง

ES6 (ECMAScript 6)
        มันคือภาษา JavaScript เวอร์ชันใหม่ที่มีฟีเจอร์มากมาย เช่น ประกาศฟังก์ชันโดยใช้ลูกศร (=>), ประกาศตัวแปรใน block scope ด้วย let, Class และอื่นๆ ซึ่งใน React สาเหตุที่ใช้ ES6 เพราะฟีเจอร์ใหม่ๆ มันช่วยให้เราเขียนโค้ดได้ง่ายขึ้น และสวยกว่าเดิมมาก เทียบกับเขียน JavaScript เดิมๆ
Babel
        สืบเนื่องจาก ES6 เนี่ย browser บางตัวยังไม่รองรับ ถ้าเราอยากเขียน ES6 เราจะต้องแปลงให้เป็น JavaScript ที่ browser รองรับได้ ซึ่ง Babel เป็นคนทำส่วนนี้ให้
Node.js
        Cross Platform Runtime Environment สำหรับฝั่ง Server และเป็น Open Source ซึ่งเขียนด้วยภาษา JavaScript สรุปรวมๆ NodeJS ก็คือ Platform ตัวหนึ่งที่เขียนด้วย JavaScript สำหรับเป็น Web Server
Command Line
        สามารถใช้ Command Line ได้ พอรู้จัก NPM, Yarn เป็นต้น
Webpack
        เป็นเครื่องมือสำหรับช่วยแปลงโค้ด ES6 เป็น ES5 หรือเวลาเราเขียนโค้ดหลายๆไฟล์ เราอาจจะประสบปัญหาหลายๆอย่าง เช่น เราจะ include script ไหนก่อนดี รวมถึงพอไฟล์เยอะๆ มันก็จะโหลดช้า ตัว Webpack ก็จะมัดรวมพวกนี้ให้เป็นไฟล์เดียว พร้อมทั้งจัดลำดับการ include ให้เลย
คอนเซปต์ที่เราต้องรู้เพื่อเขียน React หลักๆ มี 3 อย่าง
  • Component – ส่วนต่าง ๆ ในเว็บเราจะมองเป็น Component

  • State – ข้อมูลที่อยู่ใน Component แต่ละชิ้น เราเรียกว่า State

  • Props – ข้อมูลที่ถูกส่งต่อจาก Component ชั้นบนลงไปชั้นล่าง เราเรียกว่า Props (Properties)

    Component คืออะไร ?

            ถ้าพูดถึงว่าเราจะใช้ React เพื่อเขียนอะไร คำตอบคือเรากำลังจะเขียน Component
            แนวคิดของ Component คือการเขียนองค์ประกอบของ View ให้ย่อยที่สุดเท่าที่จะย่อยได้ และเน้นให้ Reusable ได้เยอะ

            ตัวอย่างเช่น ถ้าเราสามารถเขียน Component ของ Searchbox ไว้เพียงครั้งเดียว และเมื่อมีใครอยากใช้ Searchbox ก็แค่ไปดึงมาใช้ ถ้าจะแก้ก็แก้ที่ Searchbox Component อันเดียว ที่เหลือจะเปลี่ยนตาม ซึ่งการแบ่ง Component จะมีข้อดี คือ

    • มีการแบ่งส่วนกันเขียน แยกส่วนกันชัดเจน

    • สามารถนำ component ไปใช้ที่อื่นได้ หากต้องการอะไรที่คล้ายๆ กัน

    • แก้ไขง่าย

    ตัวอย่าง จาก Facebook
            เราสามารถแบ่งหน้าจอนี้ออกมาได้ 5 Component ด้วยกัน คือ

    1. ProductRow (สีแดง): แสดงชื่อสินค้า และราคา

    2. ProductCategoryRow (สีฟ้าอ่อน): แสดงชื่อประเภทสินค้า

    3. ProductTable (สีเขียว): แสดงรายการสินค้า (ซึ่งจะประกอบด้วย ProductCategoryRow + ProductRow)

    4. SearchBar (สีน้ำเงิน): แสดง Search Box ที่ filter ข้อมูลตาราง พร้อมปุ่ม Option ว่าจะให้โชว์เฉพาะสินค้าที่มีใน stock

    5. FilterableProductTable (สีส้ม): แสดงข้อมูลสินค้า ที่เราสามารถ filter ข้อมูลในตารางได้ ด้วยการพิมพ์ใน search box (เป็นตัวครอบทุกอันทั้งหมด)

            โดย Component ใน React นั้นสามารถเขียนได้หลายแบบมาก แต่แบบที่ใช้กันเยอะและเป็นมาตรฐานสำหรับมือใหม่ คือ การ extends คลาส Component ของ React มานั่นเอง



            เวลาข้างนอกจะเอาไปใช้ ก็จะใช้เหมือนเรามี tag HTML นั้นๆมาให้ใช้เลย


    Props กับ State คืออะไร ?

            ให้นึกถึง HTML ธรรมดา อย่างใน tag a เราสามารถใช้ href เพื่อบอกได้ว่ากดลิ้งแล้วจะไปไหน โดยการเขียนต่อไปใน tag a เช่น


            ซึ่งใน React การทำ href ก็ทำได้เหมือนกัน โดยเราจะเรียกว่า props นั่นเอง (มาจาก properties) props เป็นจุดที่ทำให้ React component นึง สามารถ pass ข้อมูลต่างๆ ไปอีก component นึงได้

    จากรูปจะเห็นได้ว่า

    • ParentComponent render ChildComponent โดยมี props message=“Hello World” ไปด้วย

    • ChildComponent จะ render คำว่า And then I said, “Hello World” ให้ เพราะ ParentComponent ส่งคำว่า Hello World ผ่าน props ที่ชื่อ message นั่นเอง

            ด้วยเหตุนี้ ทำให้ React นั้นมีการไหลของข้อมูลเป็นการไหลไปทางเดียว คือ จากพ่อสู่ลูก นอกจากนี้ props ไม่จำเป็นต้องเป็นแค่ค่าตัวเลข หรือ string ธรรมดา มันอาจจะเป็น Object ทั้งก้อน หรือฟังก์ชันก็ได้ ซึ่งแน่นอนว่ามีแค่ props คงไม่พอ บาง Component ก็อยากจะต้องมีการสร้าง data อะไรบางอย่าง ไว้ใช้ในตัวมันเอง หรือส่งไปให้ลูกๆ เช่นกัน และนี่ทำให้

    "state"

    เข้ามามีบทบาท



        state

    นั้นเสมือนเป็น data ที่มีการใช้แค่ภายใน Component นั้นๆ อารมณ์คล้ายกับเป็น local variable ซึ่งการกระทำแก้ไขข้อมูลของ state นั้นจะทำผ่านฟังก์ชัน setState ของ React และ state นั้นจะเป็น JavaScript object ธรรมดา ดังนั้นหน้าตามันจะเป็นยังไงขึ้นอยู่กับว่าเราจะสร้างสรรค์ออกมายังไง





    จากรูปข้างบนนี้ ถ้ากดปุ่ม ก็จะมีการ set state ให้มีค่าการถูกกด (clicked) เป็น true


            ส่วนรูปข้างล่างนี้ เป็นตัวอย่างแสดงให้เห็นภาพว่า state เอาไปใช้ได้ยังไง โดยจะทำตัว CounterComponent ที่จะมีปุ่มกดที่จะเพิ่มเลขได้ และค่าเลขปัจจุบันก็จะถูกแสดงออกมา


    ลำดับการทำงาน

    • constructor จะใส่ค่า state ให้มี count เป็น 0

    • ประกาศฟังก์ชัน handleClick ที่จะรอ event การกด หากมีการกด จะ set state ใหม่โดยให้ค่า count เพิ่มขึ้นจากเดิมไป 1

    • render() จะ render component มา โดยจะแสดงจำนวนเลขปัจจุบัน และมีปุ่ม

    • โดยเลขปัจจุบัน จะแสดงผ่านตัวแปร this.state.count (ถ้าไล่ดีๆ มันก็คือไล่ JavaScript object ธรรมดาเลย ไม่มีอะไรแปลก)

    • และปุ่มจะผูกกับฟังก์ชัน handleClick ผ่าน onClick

    • ดังนั้นเมื่อกดปุ่ม ก็จะเรียกฟังก์ชัน handleClick ที่จะเพิ่มค่า count ไป 1 ผ่านการ set state

    • และ view จะอัพเดตหน้าตา เมื่อ state เปลี่ยน

    แล้วเราสามารถโยน state เข้า props ของ component ลูกได้มั้ย ?

    ตอบ : ได้ แต่ตัวลูกไม่สามารถแก้ไข state ของตัวพ่อได้ เพราะมันเป็นแค่ props ตัวนึง


            แม้ว่า React จะมี state ให้เราเพื่อที่เราจะโยนของลงไปให้ Component ต่างๆ แล้ว แต่ถ้าแอพเริ่มใหญ่ขึ้นการจัดการ state ก็จะยากขึ้นและงงมากขึ้น จึงมี architecture ต่างๆ มาช่วยให้เราจัดการ state ได้ง่ายขึ้น เช่น Flux, Redux, MobX เป็นต้น



    Flux คืออะไร ?

            คือ Data Flow Pattern เเบบหนึ่งที่ถูกเเบบโดยทีมผู้พัฒนา Facebook โดยเจ้า Flux ออกเเบบมาเพื่อใช้เเก้ไขปัญหาเรื่องความซับซ้อนในการเขียนโค้ด Flux เเละในองค์กรต่าง ๆ มักจะหยิบ Flux มาใช้ควบคู่กับ React ค่อนข้างบ่อยมาก

            โดย Flux นั้นออกเเบบมาเพื่อเเก้ไขปัญหาของเจ้าใหญ่อย่าง MVC framework เนื่องจากปัญหาของ MVC นั้นคือ กรณีที่ Project เริ่มใหญ่ขึ้น ความซับซ้อนก็เริ่มมากขึ้น ส่วนของ Model เยอะตามการเข้ามาเเก้ไขก็ลำบากขึ้น ดังนั้นเเล้ว Flux จึงออกเเบบให้กลายเป็น Unidirectional นั่นเอง

            คือ State Management (ตัวจัดการ State) นั่นเอง และมันก็ไม่จำเป็นว่าจะต้องใช้กับ React เท่านั้น มันเป็น Concept ฉะนั้น มันสามารถประยุกต์ใช้ได้หมด เช่น Angular หรือ Vue

    Stateless Component คืออะไร ?


            Stateless Component คือ Component ที่ไม่มีการใช้ State ใดๆ ใช้แต่ props เท่านั้น และเขียนมันออกมาในลักษณะการเขียน function ที่ return HTML ไป




            ซึ่งการเขียน Stateless Component เหมาะสำหรับกรณีที่เราไม่ต้องการ state เลย ไม่ต้องทำ lifecycle hook และการทำให้กลายเป็น Presentational Component ของจริง คือมีหน้าที่โชว์อย่างเดียว เราจะรู้เลยว่าถ้า logic จะมีปัญหา มันก็จะมาจากที่อื่น ไม่น่าจะมาจากที่นี่ เพราะมันโชว์เป็นอย่างเดียว (หรือก่อให้เกิดการเรียก action อย่างเดียว หากส่ง function ลงมา)



    JSX คืออะไร ?


            ถูกพัฒนาให้เป็นส่วนเสริมของ JavaScript ที่จะช่วยให้เราจัดการกับการเขียน UI (HTML) ให้ถูก render ภายใน React app ได้ง่ายขึ้นและด้วยการที่มันพัฒนาจากพื้นฐานของ JavaScript ทำให้เราสามารถใช้ความสามารถของ JS อย่างเต็มรูปแบบมาช่วยในการคำนวณ จัดการข้อมูลและเหตุการณ์ต่างๆเพื่อให้ UI จะถูก render ตามความต้องการของเราได้

            สรุปง่ายๆคือ มันจะช่วยแฝง JavaScript ไปใน HTML นั่นเอง เช่น



    จากรูปจะเห็นว่า เราสามารถ wrap โค้ด JavaScript หรือตัวแปร ไว้ใน HTML ได้ ด้วย { } นั่นเอง นั่นทำให้เราสามารถเขียนในรูปแบบที่ยากขึ้นไปได้อีก เช่น ใช้ conditional operator



    หรือจะสร้าง element ซ้ำๆ จาก array โดยใช้ map()






    React Lifecycle

            การเกิดและดับไปของ Component ใน React นั้นเราจะเรียกมันว่า Lifecycle ซึ่งถ้าเราเข้าใจส่วนนี้ได้ เราก็สามารถแทรกโค้ดเพื่อให้ถูกรันเฉพาะเมื่อเกิดในจังหวะต่างๆใน cycle ดังกล่าวได้ โดยการแทรกโค้ดนั้น เพียงเขียนฟังก์ชันตามชื่อที่มีอยู่ใน Lifecycle ใน component นั้นเป็นอันจบ


            จากรูปข้างบนนี้ อธิบายได้ว่า

    1. ครั้งแรก component จะโดน render ผ่านฟังก์ชัน ReactDOM.render() ที่จะ render component ทั้งหมด หากใน component มี component ย่อย ก็จะ render อันย่อยด้วย ไล่ render ไป

    2. จังหวะแรก จะมีการ getDefaultProps() ที่จะดึงค่า props ที่เป็น default มาก่อนทั้งหมด

    3. และก็จะ getDefaultState() เหมือนกัน

    4. ต่อมาจะเป็นจุดสำคัญ คือในจังหวะที่ตัว DOM ที่กำลังจะ render มันจะถูกแปะใน DOM หลัก มันคือจังหวะของ componentWillMount() ซึ่งเราสามารถมา hook cycle นี้เพื่อทำอะไรบางอย่างได้ เช่น เริ่มยิง AJAX ขอข้อมูลจาก API เป็นต้น

    5. ถัดจากนั้น เมื่อ DOM ถูกแปะใน DOM หลักแล้ว ก็จะทำการ render() ตัว DOM ขึ้นมา ตามที่เราเขียนไป

    6. เมื่อ render() เสร็จ ตอนนี้ DOM นั้นก็จะแปะอยู่ในเว็บอย่างสมบูรณ์แล้ว ก็จะเป็นจังหวะของ componentDidMount() ที่เราสามารถแตะต้อง DOM ที่เราเขียนได้แล้ว เช่น หากเราอยากใช้ library ของ jQuery เราก็มาใช้มันในนี้เพราะ DOM จริงๆ มันถือกำเนิดแล้ว (ก่อนหน้านั้น DOM มันกำลังจะเกิดเฉยๆ)

    7. หลังจากนั้น อาจจะเกิดเหตุการณ์เช่น props จากด้านบนเปลี่ยนค่า หรือเปลี่ยน props ผ่าน setProps() จังหวะที่ props เปลี่ยนนั้นก็จะเป็นช่วงของ componentWillReceiveProps(nextProps) คือเป็นจังหวะที่ Component จะได้รับ props อันใหม่

    8. หรือหากไม่เกิดการเซ็ต props ใหม่ แต่เกิดการเซ็ต state ผ่าน setState() จังหวะนั้นก็จะกระโดดมาที่ shouldComponentUpdate(nextProps, nextState) ส่วนนี้เป็นส่วนที่สำคัญมาก เพราะว่า หาก props เปลี่ยนหรือ state เปลี่ยน มันจะโดนมาอยู่ที่จุดนี้ ซึ่งเป็นจุดที่ Virtual DOM ของ React มีบทบาทมาก และความเร็วของมันเกิดจากจุดนี้ คือมันจะประเมินว่า component นี้ต้องมีการอัพเดตหน้า view หรือไม่ โดยมีการใช้ Virtual DOM ในการดู

    9. หาก Virtual DOM ตัดสินใจจะอัพเดต view ละ ก็จะไปที่ componentWillUpdate(nextProps, nextState) เลย

    10. และก็จะสั่ง render() อีกครั้ง เพื่อ rerender ใหม่

    11. และไปที่ componentDidUpdate(prevProps, prevState) เพื่อบอกว่าการอัพเดต component เสร็จแล้ว

    12. แล้วก็จะกลับไปสู่จุดรอ และเป็นวังวนไป หากมี state หรือ props เปลี่ยน ก็จะไปทำตามข้อ 7 หรือ 8 ใหม่

    13. หาก component นี้เกิดต้องจากลา ก็จะไปสู่ componentWillUnmount() ที่จะทำการ unmount DOM นี้ออกจาก DOM หลักออกไป แล้วก็ตายจากไป

            การทำความรู้จักว่า Lifecycle มีอะไรบ้างนั้นช่วยให้เราสามารถเขียนโค้ดไป hook (เกาะ, เกี่ยว) กับช่วง Lifecycle ที่ต้องการได้ เช่น อยากจะยิง API ขอข้อมูล ก็ไปยิงตั้งแต่จังหวะที่จะโดน mount เลยสิ (componentWillMount()) หรือจะต้อง init Google Map ก็ต้องรอให้ DOM จริงๆ เกิดขึ้นก่อน ถึงจะ init ได้ (componentDidMount()) อะไรแบบนี้ หรือบางเคส เราตัดสินใจแทน Virtual DOM ได้ว่า ถ้าอีค่านี้เปลี่ยน ยังไง view ก็ต้องเปลี่ยน ก็ไปเขียน hook shouldComponentUpdate() แล้วตัดสินใจให้เลยก็ได้ ก็จะ optimize ประสิทธิภาพไปได้อีก






    สมองของ React ทำมาจากอะไร


            สงสัยกันไหม ว่าทำไม React ถึงทำงานได้เร็วมากๆ นั่นก็เพราะตัว React นั้นมีการเปรียบเทียบการเปลี่ยนแปลงของ View จากการใช้ Virtual DOM

            อย่างที่ทราบกันว่า เวลาเรามี HTML มา Browser ก็จะ render มันออกมาให้เรา แต่ว่าตัว element ต่างๆ ใน HTML นั้นสามารถถูก access ได้เพื่อทำอะไรบางอย่าง เช่น CSS มา access เพื่อ styling ให้มัน, JavaScript มา access เพื่อแปะ event ให้มัน ดังนั้น ตัว Browser มันจะไม่ได้แค่ render HTML ออกมาเป็นหน้าตาเฉยๆ แต่ยังแปลงตัว HTML ที่เราเขียนเนี่ย ไปเป็นต้นไม้ต้นนึง ที่เราเรียกมันว่า DOM (Document Object Model) Tree




            ตัว DOM ไม่ได้ทำงานเร็วเสมอไปทุกครั้ง สมมติถ้าหน้า View เกิดมีการอัพเดต เช่น มี element โดนถอนออก หรือมีการแก้ค่าข้างใน DOM ก็จะต้องแก้ต้นไม้ใน node ที่ต้องการอัพเดต แม้ว่า DOM มันจะหา node นั้นได้เร็วเพราะเป็นต้นไม้ แต่หากมีการอัพเดตแล้ว node นั้นมีลูกเยอะๆ ก็จะส่งผลกระทบหนักมาก ลองนึกภาพว่าถ้าขยับ node นึง 10 pixel แต่จริงๆ แล้วทุก node ลูกนั้นๆ ต้องขยับตามด้วย และดันมีเป็นพันๆ node แน่นอนว่าจะส่งผลให้การทำงานมันจะช้ามาก เพราะต้องลงไปไล่แก้

            อีกทั้ง DOM นั้นก็ไม่เก่งเรื่องการสร้าง Dynamic View ที่มีการปรับเปลี่ยนค่าใน DOM ตลอดด้วย ทำให้เมื่อหน้า View เริ่มมีความ dynamic สูงขึ้น โดยเฉพาะใน SPA ทั้งหลาย DOM จึงส่อถึงปัญหาของประสิทธิภาพอย่างชัดเจนมากขึ้น ด้วยปัญหาเหล่านี้เลยมีการเสนอแนวคิดนึงว่า เรียกว่า

    "Virtual DOM"

    ขึ้นมา



            Virtual DOM บอกว่า เราจะ copy DOM ทั้งต้นมาด้วยเลย โดยเรียกมันว่า Virtual DOM เวลาที่ View อยากจะอัพเดตอะไรก็ตาม มันจะมาทำใน Virtual DOM ก่อน เมื่อแก้ใน Virtual DOM เสร็จ หากพบว่ามี node ที่ View ต้องเปลี่ยนจริงๆ ก็จะทำการ patch DOM จริงๆ ให้ แต่คราวนี้เขาจะรู้ละว่ามี node ไหนที่ต้อง patch จริงๆ เพราะ View เปลี่ยน ก็จะไป patch DOM จริงเฉพาะที่จะเปลี่ยน หน้าตาก็จะเปลี่ยน ซึ่งมันจะมี algorithm ในการตรวจเช็คว่ามีจุดไหนใน DOM จริงกับ Virtual DOM บ้างที่แตกต่าง



            Virtual DOM ด้านบนจะประเมินว่า node ไหน view มีการอัพเดตบ้าง แล้วจึงเทียบกับ DOM จริงๆ แล้วทำการอัพเดตให้ตรงกัน ซึ่งผลของการทำ Virtual DOM มีข้อดี คือเราสามารถ specific จุดที่ต้องแก้ไขได้เลยจริงๆ เพราะเราประเมินมาชั้นนึงแล้วว่ามีส่วนไหนต้องอัพเดตบ้าง ทำให้ไม่เสียเวลากับการที่ต้องแก้ DOM จุดที่ไม่จำเป็นต้องแก้จริงๆ นั่นทำให้ React จึงโดดเด่นเรื่องประสิทธิภาพเอาเรื่อง เพราะมีการใช้ Virtual DOM นี่แหละ






    ข้อดีและข้อเสียของ React


    ข้อดีของ React

    • เทรนด์เว็บไซต์เริ่มกลายเป็น API – เว็บไซต์ในปัจจุบันมี API กันเยอะมาก (เว็บใหญ่ ๆ เช่น Facebook, Google, Twitter etc. ก็มี API ครบกันหมดแล้ว) เพราะฉะนั้นถ้าวันนี้เรายังทำเว็บที่คุยกับ API ไม่ได้ เราจะเสียเปรียบมากในอนาคตครับ
    • Concept เข้าใจง่าย – เรียนรู้ตัว React เองง่ายมาก และเครื่องมือที่ช่วยเรียนรู้เยอะมาก ๆ
    • อนาคตไกล – React เกิดมาหลายปีแล้ว และยังคงมีคนให้ความสนใจเยอะ เพราะฉะนั้นเรียนตอนนี้ยังใช้ได้อีกหลายปีแน่นอน
    • เครื่องมือทำงานด้วยเยอะ – หลายคนอาจจะกลัวว่าเราต้องศึกษาเครื่องมือเพิ่มเติมมากมายเพื่อใช้ React แบบเต็มที่ แต่จริง ๆ เรียนรู้แค่ React ก็ทำเว็บได้แล้ว
    • ทำแอพมือถือได้ด้วย – เครื่องมือหนึ่งที่ทำให้ผมมาสนใจ React คือ React Native ซึ่งเป็นการเขียน JavaScript แล้วนำมาแปลงเป็น App แบบ Native ได้ทั้งบน Android และ iOS เครื่องมือนี้พัฒนาโดยทีม Facebook เองเลย
    • คนใช้เยอะ หาโค้ดให้ศึกษาง่าย – มี Best Practice จากเว็บไซต์จริงเยอะแยะมากมายให้ศึกษา เพราะเว็บใหญ่ก็ใช้กันเยอะ ลองดูได้ที่ ลิสต์เว็บที่ใช้ React บางส่วน (ย้ำว่าบางส่วนนะครับ)
    • Community แข็งแกร่งมาก – ถ้ามีคำถามไม่ต้องกลัวว่าจะไม่มีคนตอบ เพราะกระแสแรงทั้งในไทย (ReactJS Developers Thailand ซึ่งเป็นทีมงานเดียวกับที่จัดงาน React Bangkok) และต่างประเทศ ล่าสุดผมกดบัตร Meetup React Melbourne ไม่ทัน เพราะประกาศแปปเดียวยอดคนเข้าไป 200+ แล้ว


    ข้อเสียของ React

    • Documentation ยังไม่ดี – Documentation บนเว็บหลักแอบอ่านยากไปหน่อย แต่เดี๋ยวจะแนะนำเว็บที่อธิบายง่าย ๆ ที่ผมใช้ตอนเรียน React
    • ต้องรู้ JavaScript ในระดับหนึ่ง – คนที่มาจากภาษาอื่นเช่น PHP อาจจะต้องมาเรียนรู้ในส่วนนี้






    React เหมาะกับงานแบบไหน


    • งานที่มีโค้ดซับซ้อน React สามารถแก้ไขปัญหาตรงนี้ได้ เนื่องจากหั่นเป็นท่อนๆ ทำให้ง่ายต่อการจัดการ
    • งานที่ต้องการโค้ดที่จัดการได้ง่าย และมีการแชร์ Component กันหลายจุด – เพราะ React แบ่งโค้ดออกเป็นส่วน ๆ ดูเป็นระเบียบมาก
    • งานที่เผื่อต่อยอดไปสู่ Mobile App
    • ทีมงานที่มีความรู้เรื่อง JavaScript อยู่แล้วก็สามารถปรับเปลี่ยนได้เลย
    • งานที่ไม่ต้องใช้ IE8 เพราะ React เลิกรองรับ IE8 ไปแล้ว

            ผู้เป็นเจ้าตลาดเครื่องมือในการบริหารงานเพิ่ม Productivity ยอดนิยม เช่น JIRA, Confluence, Bitbucket, Trello, Hipchat ซึ่งแน่นอนว่า App เหล่านี้ก็ถูกสร้างด้วย React ด้วยเช่นกัน โดยที่ Atlassian เลือกใช้ React ในการพัฒนาเพราะ React นั้นสามารถพัฒนาได้ง่าย แถมยังรองรับได้หลายอุปกรณ์ เช่น ไม่ว่าจะเป็นบนโทรศัพท์ หรือคอมพิวเตอร์ก็ใช้งานได้

            แอปพลิเคชันสั่งอาหารเจ้าแรกๆ ของโลกที่ได้รับความนิยมอย่างมากในอเมริกา ได้เลือกใช้ React เพื่อพัฒนาเว็บไซต์สำหรับการหาร้านอาหาร, สั่งอาหาร รวมถึงการ Track Order ต่างๆ ส่วนเหตุผลที่ UberEats เลือกใช้ React ก็เพราะสามารถรองรับการใช้งานได้บน Smartphones และ Tablets ซึ่งทำให้ลูกค้า ร้านอาหาร และไรเดอร์ใช้งานได้อย่างสะดวก และมีประสบการณ์ที่ดีในการใช้งานนั่นเอง

            เจ้าตลาด Video Streaming เลือกใช้ React ในการพัฒนาเว็บไซต์เพื่อ Redesign User Interface ตั้งแต่ปี 2015 โดยสาเหตุที่ Netflix ตัดสินใจเปลี่ยนมาใช้ React เนื่องจากประสิทธิภาพของเว็บที่ได้, ความเร็วในการปล่อย Software, การสามารถแบ่งเป็น Module ย่อยๆในการพัฒนา โดยยังสามารถมี Feature รองรับการเล่นวีดิโอแบบ 4K, HDR และ Dolby Vision

    Airbnb


            แอปจองที่พักยอดนิยมที่ให้บริการทั่วโลก นอกจากใช้ React ในการพัฒนาเว็บไซต์แล้วยังได้สร้าง Tools ที่ชื่อว่า Painting with Code – Airbnb Design ซึ่งเป็น Tools ที่ช่วยให้ การทำงาน ระหว่าง Designer และ Programmer ทำได้สะดวกมากยิ่งขึ้น แถมยังช่วยให้ Community ของ React เติบโตด้วย

    Dropbox


            เว็บฝากไฟล์เจ้าแรกบนโลกที่ก่อตั้ง และดำเนินกิจการมาตั้งแต่ปี 2012 แต่ก็เจอเข้ากับปัญหาในการพัฒนาเว็บไซต์ จนกระทั่งในปี 2016 Dropbox เลือกที่จะปรับ Design Library โดยใช้ React ทำ Functions ต่างๆ เพื่อรองรับการทำงาน เช่น การเก็บรวบรวมและการคำนวณขนาดไฟล์ต่างๆ นอกจากนี้ยังมี Features ในการ Show Page ตอนที่ Scroll Down ต่างๆ นอกจากนี้ทีมงาน Dropbox ยังช่วยพัฒนา React และ JS Tools อื่นๆ เพื่อให้สามารถทำงานได้ดีขึ้นบน Web Browser ต่างๆ

    Codecademy


            บริษัทที่สอนการเขียน Code ด้วยภาษาต่างๆ เช่น Java, Python, SQL, C ++ เองก็ใช้ react ในการช่วยพัฒนาส่วนต่างๆ เนื่องจาก ใช้งานได้ง่าย ช่วยให้เว็บติด SEO ได้ง่าย นอกจากนี้ยังมีความยืดหยุ่นสูงในการพัฒนาอีกด้วย
