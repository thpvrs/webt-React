ก่อนที่เราจะไปทำความรู้จักกับ REACT ว่ามันคืออะไร เราจะพามาดูกันก่อนว่า
การทำเว็บแบบเดิมที่เราทำมันต่างจากการทำเว็บแบบใหม่ด้วย React ยังไงค่ะ

การทำเว็บไซต์แบบดั้งเดิม

        การทำเว็บไซต์ด้วยเทคนิคเดิม คือ HTML, CSS, JavaScript, PHP (หรือภาษา Backend ภาษาใดภาษาหนึ่ง) ส่วนใหญ่ “การทำงานทั้งหมดจะอยู่ที่ Server” สามารถสรุปลำดับการทำงานได้ดังนี้

1. User เข้าเว็บไซต์เรา https://www.designil.com ก็จะส่งไปบอก Server
2. Server (PHP) จะประมวลผลผ่านภาษา Backend ว่า User เข้าหน้านี้แล้วต้องให้อะไรบ้าง เช่น ต้องการบทความ 10 บทความล่าสุด
3. Server นำข้อมูลที่เก็บไว้ออกมาจาก Database (เช่น MySQL) แล้ววางลงไปในเทมเพลต HTML ที่เตรียมไว้
4. Server ส่ง HTML กลับไปหา User

การทำเว็บไซต์แบบใหม่

         การทำเว็บไซต์แบบใหม่ คือ ใช้ JavaScript ส่ง Request ไปขอข้อมูลจาก Server แล้ว Server ตอบกลับมาเป็นข้อมูลอย่างเดียว เราค่อยใช้ JavaScript เอาข้อมูลมาแสดงอีกที อย่างไรก็ตาม พอข้อมูลถูกส่งจาก Server มาให้ Web Browser แล้ว ก็เกิดปัญหาว่า “แล้วจะจัดการข้อมูลบน Web Browser อย่างไร” ซึ่งเมื่อก่อนไม่มี Library คอยช่วยเหลือด้านการแสดงผล เลยทำให้โค้ดพันกันมั่วไปหมด
        JavaScript Framework สมัยใหม่ เช่น React ก็เลยออกมาเพื่อช่วยให้เราจัดการเรื่องการดึงข้อมูล การแสดงผลข้อมูลในเว็บให้ง่ายขึ้น

1. User เข้าเว็บไซต์เรา
2. โหลด HTML, CSS, JS ขึ้นมาก่อน แล้ว JavaScript ค่อยขอบทความล่าสุดจาก Server
3. Server ได้รับคำขอ ก็ส่งไปขอข้อมูลจากฐานข้อมูลอีกที

4. Database ส่งข้อมูลกลับมาให้ Server
5. Server ส่งข้อมูลกลับไปให้ Browser ทาง API โดยส่วนใหญ่จะส่งเป็น JSON เพราะ JavaScript สามารถนำไปใช้งานต่อได้ง่าย
6. JavaScript นำข้อมูลที่ได้รับมาจัดการแสดงผลบนหน้าเว็บไซต์






                 React เป็น JavaScript library หรือจะเรียกว่าเป็น Javascript Framework ที่ใช้สำหรับสร้าง user interface ที่ให้เราสามารถเขียนโค้ดในการสร้าง UI ที่มีความซับซ้อนแบ่งเป็นส่วนเล็กๆออกจากกันได้ ซึ่งแต่ละส่วนสามารถแยกการทำงานออกจากกันได้อย่างอิสระ และทำให้สามารถนำชิ้นส่วน UI เหล่านั้นไปใช้ซ้ำได้อีก หรือจะพูดว่าเราใช้ React เพื่อทำให้เราสามารถสร้าง
